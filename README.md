# 研究助手 Agent

## 项目背景

在日常工作中，我经常需要：搜索某个话题的最新信息、整理分析、
然后生成报告。这个流程通常需要打开浏览器、复制粘贴、手动整理
——重复且低效。

研究助手 Agent 的目标是：用户只需用自然语言描述需求，
Agent 自动完成搜索、分析、保存的全流程。

## 功能

### 本次实现（MVP）
用户使用自然语言描述一个研究任务，Agent 自动完成以下操作：

- [x] 自动搜索网页获取最新信息
- [x] 读取本地文件作为分析素材
- [x] 执行 Python 代码进行数据处理
- [x] 将分析结果保存到文件

### 未来迭代
- [ ] 支持 PDF 文件读取
- [ ] 支持多轮对话追问

## 重构计划

### 当前状态
- 单文件架构（main.py ~750 行），所有代码混在一起
- 功能已实现，待优化代码结构

### 重构目标
1. **模块化重构**：按功能拆分到不同目录
2. **外观模式**：对外提供简洁接口
3. **依赖倒置**：依赖抽象而非具体实现

### 具体优化项

| 优化项 | 优先级 | 说明 |
|--------|--------|------|
| 配置管理分离 | 高 | 封装 Config 类，统一管理环境变量 |
| 日志系统 | 高 | 替代 print，使用 logging 模块 |
| 统一错误处理 | 中 | 定义 ToolResult 类，规范返回格式 |
| 工具注册机制 | 中 | 装饰器自动注册，减少手动维护 |
| 类型提示完善 | 中 | 添加类型注解，配合 mypy 检查 |
| 单元测试分离 | 低 | tests/ 目录，使用 pytest |
| CLI 入口独立 | 低 | main.py 只保留入口逻辑 |

### 目标目录结构

```
research-agent/
├── src/
│   ├── config/           # 配置管理
│   ├── tools/
│   │   ├── base.py       # 抽象接口
│   │   ├── search/       # 搜索实现
│   │   ├── file/         # 文件操作
│   │   └── execute/      # 代码执行
│   ├── agent/            # Agent 核心逻辑
│   └── api.py            # 外观接口
├── tests/                # 单元测试
└── main.py               # CLI 入口
```
